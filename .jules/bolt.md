## 2024-07-24 - O(N*M) Serial Grouping Anti-Pattern in UI Renders
**Learning:** Found a critical performance bottleneck in `app.js` where UI render functions (`renderBoxCards`, `renderOutboundBoxCards`) and compaction functions repeatedly used `Array.prototype.filter` and `Array.prototype.includes`/`indexOf` on large arrays inside `forEach` loops over products. This O(N*M) pattern blocks the main thread during high-volume scanning rounds.
**Action:** When filtering a large dataset multiple times based on a key (e.g., `itemName`), always group the dataset upfront using a `Map` (O(N) grouping, O(1) lookup). Use `Set` for fast uniqueness checks instead of arrays with `.includes`.
## 2024-07-24 - Avoid Temporal Dead Zone with Shared State in Vanilla JS Closures
 **Learning:** When adding caching variables accessed by multiple global functions or asynchronous callbacks (like Firebase event listeners), declaring `let` variables deep in the file can cause `ReferenceError` (Temporal Dead Zone) if accessed before initialization.
 **Action:** Always hoist shared state variables (like `let cachedProductWeights = null;`) to the top of the relevant scope or file to ensure they are initialized before any closures can be invoked. Additionally, always return a shallow/deep clone (e.g., `{...cachedData}`) from getter functions when returning cached mutable state to prevent side effects from caller mutations.
